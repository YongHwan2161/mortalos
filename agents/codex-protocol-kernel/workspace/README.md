# Draft workspace

Use this directory for durable protocol proposals, decision records, and test plans
that are not yet ready for shared repository paths.

- Name drafts `YYYY-MM-DD-<topic>.md`.
- Mark assumptions, unresolved questions, and promotion criteria explicitly.
- Do not duplicate source files here.
- Put disposable builds, logs, downloaded dependencies, and generated artifacts in
  a temporary directory outside the repository.
- When a draft is accepted, move the normative result into the appropriate shared
  specification or code path through a reviewed pull request, and leave only a
  short pointer here if continued context is useful.
